2.1 SAFEノートとは 2.1 What is a SAFE Note

Simple Agreement for Future Equity（将来しょうらい株式かぶしき取得しゅとく契約けいやく） Simple Agreement for Future Equity

投資家資金を提供
→資金
SAFEノート株式取得の権利
（株式そのものではない）
⇢将来転換
株式所定タイミングで転換

基本情報

  • 米国シリコンバレー発祥、日本では J-KISS として VC 投資の標準
  • 「今資金を預かり、将来の所定タイミングで株式に転換」する契約
  • 投資時点では株式ではなく「株式取得の権利」を保有
  • 採用理由：企業価値の確定を遅らせ迅速な手続きが可能

早期投資家のメリット

  • バリュエーションキャップ：将来の評価額に上限を設定 → 早期ほど有利
  • ディスカウント：将来ラウンドの価格から割引適用 → 早期ほど多く株式取得
  • 先行リスクへのリターンを契約に組み込み、後発投資家との公平性を確保

重要 金商法49人ルールは完全に適用

  • SAFEノートは金商法上「みなし有価証券」に該当
  • 株式ではなく権利でも、実質的な有価証券として規制対象
  • 1オファリングあたり日本人投資家 49名 の上限がそのまま適用

「SAFEなら49人ルールを回避できる」という誤解は誤り。
→ 50名以上は公募登録が必要（5,000万〜2億円のコスト）

レイヤー間のフロー

  • 投資会社 → SPV：資金提供
  • プロジェクト管理 → SPV：建設業務 ／ SPV → プロジェクト管理：建設委託料
  • アセット管理 → SPV：運営業務 ／ SPV → アセット管理：運営委託料

なぜこの 4 レイヤー構造か

  • ① リスク分離：各 SPV が物件・契約・負債を抱える。プロジェクト 2 の問題が プロジェクト 1 投資家へ波及しない
  • ② 税効率：建設管理料・運営管理料を別エンティティで受領 → UAE 法人税（9%）の最適化
  • ③ スケーラビリティ：同じレイヤー構造のまま プロジェクト 2、3、4 を SPV 追加で横展開可能
  • ④ 投資家への明瞭性：各 SPV は単一プロジェクトの P&L のみを抱える → 投資家が個別に評価・選択可能

各エンティティの設立コスト・CVML 業務範囲は 2.4 CVML業務範囲 ／ セクション 6 参照。

核心の問いThe core question

2 つの次元を、定款（MoA）の株式クラスの数にどう変換するか？ How do we translate 2 dimensions into the number of share classes written in the MoA?

決定 3C — 議論続行（推奨案：Model A） Decision 3C — Under discussion (recommended: Model A)

サイドレター（Side Letter）

株主契約とは 別の個別契約。特定の投資家へ追加便益を提供する仕組み。

  • 法的性質：「証券」ではない個別契約 → FIEA の株式クラス問題を引き起こさない
  • 用途：大型投資家（古竹）への VIP 待遇を、新株式クラスを作らずに実現
  • 典型的な内容：取締役オブザーバー席、四半期ブリーフィング、共同投資権、優先情報アクセス等

  • 月次 + 四半期：詳細財務 + 戦略アップデート（市場分析、ロードマップ）
  • 月次 + 四半期 + 随時：重要事象（資金調達、契約締結、リスク事案）の即時通知
  • 完全アクセス：社内データルームへの直接アクセス（高負担）

推奨古竹 = 月次+四半期+随時／林・森 = 月次+四半期／芽・根 = 月次のみ

⚠ ティアに応じて段階的に。レポート作成は実工数が発生。
共同投資権
共同投資権（Co-investment Rights）

次回ラウンド以降の 投資参加権。希薄化防止と将来資本構成の柔軟性のトレードオフ。

  • なし：次回ラウンドへの参加保証なし（標準）
  • 次ラウンド按分（pro-rata）：現持分比率に応じて次ラウンドに参加できる権利
  • 按分 + 優先閲覧：按分権 + 次ラウンド提案を他者より先に見れる権利
  • 完全先取権（preemption）：次ラウンドを優先的に取得できる強力な権利（最高リスク）

推奨古竹 = 按分+優先閲覧／林・森 = 次ラウンド按分／芽・根 = なし

⚠ 全ティアに付与すると、次回ラウンド設計が大きく制約される。
